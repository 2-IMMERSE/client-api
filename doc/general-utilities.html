General utilities

The client-api includes various general utilities which are useful for development of components, DMApps and the client-api in general.

Information on communications, signalling, ref-counting, etc. mechanisms is in Intra-client communications, signalling, and data: strategies and suggestions documentation, rather than this document.

General information on component development is in the General component documentation, rather than this document.

General information on component parameters is in the Component parameters documentation, rather than this document.

Full details are included in the relevant documentation, rather than this document.

This document is not exhaustive.

Controller

  • DMAppController: Top-level controller class which is the "entry-point" for running instances of the client-api, this has various documented options, methods and members which are worth noting.

Logging

  • Logger: Logger class, components and other entities are allocated these already.
  • See also: component logger instance, createNamedLogger.

Network communication

  • AjaxPromise: Used for making network requests (e.g. HTTP), includes logging, auto-retry, signal-interaction, etc.
  • Use ajaxPromise or ajaxPromiseNX to get an instance with sensible defaults.

Control flow

  • ExecValve: Blockable utility wrapper to queue or exec functions.
  • PromiseExecQueue: Blockable queue for serialised execution of job functions.
  • Blockable, BlockableWrapper: Useful interfaces related to blockables, see also BlockCountSignal in Intra-client communications, signalling, and data: strategies and suggestions documentation.
  • RefCountedDelayedDestructor: Reference counter with delayed destructor semantics.
  • retryPromise: Utilitity to auto-retry a promise-returning job function.
  • waitable: Promise wrapper utility with external resolve/reject methods.
  • PromiseUtil: Promise utilities: Promise.all() for objects.
  • UpdateUtil: Utilities for updates in async contexts (perform update call(s) when something is in a writable/ready state).
  • See Intra-client communications, signalling, and data: strategies and suggestions documentation in general.

Clock management

  • See various members and methods of DMAppTimeline.
  • ClockArrayIntervalScheduler: Clock array interval scheduler
  • Low-level media/clock sync tools in: MediaElementClockSource, MediaSynchroniser.
  • Miscellaneous low-level utilities in: ClockSyncUtil and UpdateSharedStateClockProperty.

Components

  • DMAppComponent: DMApp component interface, this has many documented methods, members, etc. which are worth noting.
  • DMAppComponentWrapper: Wrap a generic HTML element as a component.
  • CustomElementUtil: Utilities for custom elements/webcomponents and HTML imports.
  • See methods of DMAppLayout, ComponentContainer for various local operations on components.

Local layout

  • DMAppLayout: General local layout handling.
  • DMAppLayoutRegionCtl: Local layout region control.

Debugging

  • See component dumpDebugInfo and DebugDumper for debug dumping from components (into the debug component, for example).

Input

  • InputDocument: Input document handling and processing.
  • InputUtil: Input handling utilities: check if a string is in the list of permitted values, parse a string into a boolean.

Test interface

  • TestConfigTestComponent: Test interface component and page: parameters, config, usage, etc.

Media

  • Controller key store: Key store for encryption keys on local device.
  • DMAppAVPlayerComponent: General interface for media player components.
  • Video/audio component: General info and parameter documentation.
  • muteAll: Convenient way to mute all media components.

Error management

  • ErrorUtil: Error flags/signals, see various errorSignals and userError* members of DMAppController.

TV emulator device

  • DMAppTvEmuController: TV emulator discovery server control and app2app functionality.
  • DMAppTvEmuSync: TV emulator DVB-CSS sync master functionality.

Companion device

  • DMAppComp: Companion device module/class.
  • setupCompanionPlatformSpecificDiscovery: Method to setup discovery on companion.
  • DMAppCompDiscoveryFilter: Utility class to filter a set of discovered devices.

Versioning

  • FeatureVersionSet: Feature version set, each top-level module has one of these.

Miscellaneous

  • argCheck: Function to do basic validation on function arguments.
  • TimeoutHandler: Utility wrapper around setTimeout with cancellation.
  • EnumUtil: Utilities for handling of enumeration types.
  • MiscUtil: Various utilities.
    • Read-only wrappers
    • Object stringification helper
    • Efficient event forwarding
